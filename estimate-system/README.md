# 見積徴収システム - セットアップ手順

Google スプレッドシート + Google Apps Script + Google サイトを使った見積徴収システムです。
協力会社は **Googleアカウント不要** で見積入力ができます。

---

## 前提条件

- Google Workspace アカウント（`yuzuka.yamazaki@sgc-web.co.jp`）
- Google Drive の外部共有が許可されていること
- 「リンクを知っている全員」への共有が許可されていること

## ファイル構成

```
gas/
├── appsscript.json          # GASマニフェスト（タイムゾーン・権限）
├── Config.gs                # 設定定数・ログ機能
├── Code.gs                  # メニュー・初期セットアップ
├── EstimateOperations.gs    # 見積依頼作成・テンプレート特定・共有設定
├── TemplateManager.gs       # テンプレート一括登録・フォルダスキャン
├── PortalManager.gs         # 協力会社ポータルシート管理
├── EmailManager.gs          # 見積依頼メール・リマインドメール
└── Monitoring.gs            # ステータス監視・期限管理・自動実行
```

## セットアップ手順

### STEP 1: マスター管理スプレッドシートの作成

1. [Google Drive フォルダ](https://drive.google.com/drive/u/0/folders/0AAD_-egrQbLiUk9PVA) を開く
2. 「新規」→「Google スプレッドシート」で新しいスプレッドシートを作成
3. 名前を「**見積管理マスター**」に変更

### STEP 2: GAS スクリプトのコピー

1. スプレッドシートで「拡張機能」→「Apps Script」を開く
2. 既存の `コード.gs` の中身を全て削除
3. 以下の順番でファイルを作成（`+` ボタンで「スクリプト」を追加）：

| 作成するファイル名 | コピー元 |
|---|---|
| `Config` | `gas/Config.gs` |
| `Code` | `gas/Code.gs` |
| `EstimateOperations` | `gas/EstimateOperations.gs` |
| `TemplateManager` | `gas/TemplateManager.gs` |
| `PortalManager` | `gas/PortalManager.gs` |
| `EmailManager` | `gas/EmailManager.gs` |
| `Monitoring` | `gas/Monitoring.gs` |

4. `appsscript.json` の設定：
   - 左メニューの「プロジェクトの設定」（歯車アイコン）を開く
   - 「`appsscript.json` マニフェスト ファイルをエディタで表示する」にチェック
   - エディタに戻り、`appsscript.json` の内容を `gas/appsscript.json` で上書き

5. 「保存」ボタン（💾）をクリック

### STEP 3: 初期セットアップの実行

1. スプレッドシートに戻り、ページをリロード
2. メニューバーに「**見積管理**」メニューが表示される
3. 「見積管理」→「設定」→「**初期セットアップ**」をクリック
4. 初回実行時に権限の承認ダイアログが表示されるので「許可」する
5. 各シートが自動作成される

> **既存環境を更新する場合:** 「見積管理」→「設定」→「シート追加セットアップ（既存環境用）」を実行すると、工種マスター・テンプレート管理シートが追加されます。

### STEP 4: 設定シートの入力

「設定」シートに以下を入力：

| 設定項目 | 入力例 |
|---|---|
| 自社名 | 株式会社〇〇建設 |
| 担当者名 | 山崎 柚花 |
| 電話番号 | 03-XXXX-XXXX |
| メールアドレス | yuzuka.yamazaki@sgc-web.co.jp |

### STEP 5: 工種マスターの確認

初期セットアップで約30種の代表的な工種が自動登録されます。
必要に応じて工種の追加・削除・名前変更を行ってください。

> **重要:** 工種名はテンプレートファイル名との自動マッチングに使われます。

## テンプレートの準備と登録

### 案件ごとのテンプレート管理の考え方

```
見積テンプレート/（Drive フォルダ）
├── P001_B邸新築工事/
│   ├── 電気設備工事（スプレッドシート）
│   ├── 内装工事（スプレッドシート）
│   ├── 設備工事（スプレッドシート）
│   └── ... （案件で必要な工種分）
│
├── P002_C邸改修工事/
│   ├── 電気設備工事
│   └── ...
│
└── 共通_白紙テンプレート/
    └── 白紙見積書
```

### テンプレート登録手順

**方法1: フォルダから自動一括登録（推奨）**

1. 案件マスターに案件を登録（テンプレートフォルダID列は空欄でOK）
2. メニュー「見積管理」→「テンプレート」→「フォルダからテンプレート一括登録」
3. 案件コードを入力
4. GASがフォルダ内のファイルをスキャンし、工種マスターとファイル名を照合
5. マッチしたファイルをテンプレート管理シートに自動登録

> **自動マッチのコツ:** ファイル名に工種名を含めてください。
> 例: 「電気設備工事」「内装工事_B邸見積」「空調換気設備工事_テンプレ」

**方法2: テンプレート管理シートに手動登録**

テンプレート管理シートに直接入力：

| 案件コード | 案件名 | 工種 | テンプレートID | テンプレートURL | 登録日 |
|---|---|---|---|---|---|
| P001 | B邸新築工事 | 電気設備工事 | スプレッドシートのID | | 日付 |

### テンプレートの推奨設定

- 入力してほしいセル（単価・金額欄）を **黄色背景** に設定
- 入力不要セルに **シートの保護** を設定（「データ」→「シートと範囲を保護」）

## 使い方

### 見積依頼を出す

1. **協力会社マスター** に協力会社の情報を登録（会社コード・会社名・メールアドレス・主要工種）
2. **案件マスター** に案件情報を登録（案件コード・案件名・見積提出期限）
3. **テンプレート準備** テンプレートをフォルダに配置して一括登録
4. **見積依頼管理** に依頼を登録：
   - `B列（案件コード）` を入力
   - `D列（会社コード）` を入力
   - `F列（工種）` を入力（空欄なら会社の主要工種を使用）
   - `K列（ステータス）` を「**未依頼**」に設定
5. メニュー「見積管理」→「**見積依頼を作成**」をクリック
   - テンプレート管理から該当テンプレートを自動特定
   - テンプレートがコピーされ、共有設定され、ポータルシートが更新される
6. メニュー「見積管理」→「**見積依頼メールを送信**」をクリック

### ステータスを確認する

- メニュー「見積管理」→「**ステータスを一括更新**」

### リマインドを送る

- メニュー「見積管理」→「**リマインドメールを送信**」

### 期限切れシートをロックする

- メニュー「見積管理」→「**期限切れシートをロック**」

## Google サイトの設定（協力会社ポータル）

### サイト作成手順

1. [Google Sites](https://sites.google.com) にアクセス
2. 「新しいサイト」→「空白」で作成
3. サイト名を「**〇〇建設様 見積ポータル**」に設定
4. ページ内に以下を配置：
   - テキスト: 「〇〇建設様 専用ページ」（見出し）
   - **埋め込み**: 「挿入」→「ドライブ」→ 該当会社のポータルシートを選択
   - テキスト: 操作マニュアルリンク（任意）
5. 「公開」→ 共有設定を「リンクを知っている全員」にする
6. サイトURLを「協力会社マスター」の「サイトURL」列に記録

## 協力会社側の操作フロー

1. メールでサイトURL（またはスプレッドシートURL）を受け取る
2. URLをクリックして開く（**Googleアカウント不要**）
3. ポータルから「入力画面を開く」をクリック
4. スプレッドシートが開く → 黄色セルに単価等を入力
5. 自動保存されるのでそのまま閉じる

以上。
